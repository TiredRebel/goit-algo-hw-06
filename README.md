# Огляд

Набір скриптів для роботи з графом київського метрополітену: побудова мережі, аналіз базових метрик, пошук шляхів (DFS/BFS) та знаходження найкоротших маршрутів алгоритмом Дейкстри.

## Вимоги
- Python 3.13.2+
- Залежності: `networkx`, `matplotlib`

Встановлення залежностей:
```bash
uv sync
```

## Завдання 1: Побудова та аналіз графа
- Файл: `task1.py`
- Дії:
  - Створює неорієнтований граф ліній метро (станції, лінія, орієнтовна тривалість поїздки).
  - Обчислює базові метрики: кількість вершин/ребер, густина, середній та максимальний ступінь.
  - Візуалізує граф з кольорами ліній і підписами тривалостей, зберігає зображення та автоматично відкриває його.
- Запуск:
```bash
uv run task1.py
```
- Результат: метрики у консолі; зображення у `artifacts/kyiv_metro_graph.png` (відкриється автоматично).

## Завдання 2: DFS та BFS
Програма використовує алгоритми DFS і BFS для знаходження шляхів у графі, який було розроблено у першому завданні. Порівнює результати виконання обох алгоритмів для цього графа, висвітлює різницю в отриманих шляхах та пояснює, чому шляхи для алгоритмів саме такі.

- Файл: `task2.py`
- Запуск:
```bash
uv run task2.py
```
- Результат: виводить знайдені шляхи DFS і BFS, їх довжини та пояснення відмінностей між алгоритмами.

## Завдання 3: Алгоритм Дейкстри
- Файл: `task3.py`
- Дії:
  - Використовує ваги `duration_minutes` і рахує найкоротші шляхи між усіма парами вершин (через `nx.single_source_dijkstra`).
  - Виводить довжину маршруту (хвилини) та повний шлях для кожної пари.
- Запуск:
```bash
uv run task3.py
```
